# qmt_stock_gho

基于 QMT 迅投极速交易终端，实时镜像个人股票账户数据，实现当日委托/成交回放、持仓监控、多账户权限管理。数据可用于 AI 分析操作逻辑、一键生成公众号推文、口播稿等。适合财经博主、亲友等人群分享自己的操作逻辑和交易想法。

## 功能特性

- 实时采集账户资产、持仓、委托、成交数据，自动导出为 JSON
- 移动端友好的看板页面，支持白/暗主题
- 多密码多权限访问管理，支持资产查看、持仓查看权限独立控制
- API 接口带每日动态密钥校验，保障数据安全
- 前后端分离架构，交易端与展示端物理隔离

## 架构说明

```
┌─────────────────────┐        ┌─────────────────────┐        ┌─────────────────────┐
│   交易端 (QMT)       │  JSON  │   API 服务端         │  HTTP  │   展示端 (Flask)     │
│                     │ ─────> │                     │ <───── │                     │
│ [QMT]stock_exporter │        │      api.py         │        │      app.py         │
│  每30秒导出持仓数据   │        │  提供数据查询接口     │        │  看板页面+权限管理    │
│  → C:\stock.json    │        │  动态密钥校验        │        │  多账户权限控制       │
└─────────────────────┘        └─────────────────────┘        └─────────────────────┘
   涉及交易软件的机器                       公网服务器 / 内网穿透                     另一台安全服务器
```

## 文件结构

```
├── [QMT]stock_exporter.py   # QMT策略脚本，采集账户数据导出为JSON
├── api.py                    # 数据API服务，提供带密钥校验的数据接口
├── app.py                    # 看板Web服务，多账户权限管理+前端渲染
├── templates/
│   ├── index.html            # 看板主页面

```

## 部署步骤

### 第一步：QMT 数据导出

将 `[QMT]stock_exporter.py` 代码导入 QMT 迅投极速交易终端中，使其能够产生 `stock.json` 数据文件。

- 在迅投极速中新建策略，填入本文件内容，绑定任意股票即可运行
- 确保策略 K 线周期设置为 **1分钟** 或 tick 级别（导出间隔 30 秒）
- 数据文件默认导出到 `C:\stock.json`，可在脚本中修改 `EXPORT_DIR` 配置

### 第二步：运行 API 服务

在具备公网 IP 或内网穿透的设备上运行 `api.py`：

```bash
pip install -r requirements.txt
python api.py
```

- 默认监听 `0.0.0.0:43958`
- 接口地址：`http://你的服务器公网IP:端口号/api/chaxunData`
- 接口采用每日动态 MD5 密钥校验（规则：`s_t_o_c_k_` + 当日日期）

### 第三步：运行看板服务

在另一台不涉及交易软件的安全服务器上运行 `app.py`（同时将 `templates` 文件夹一并拷贝）：

```bash
pip install -r requirements.txt
python app.py
```

- 默认监听 `127.0.0.1:5985`
- 通过前面 API 接口进行数据转发
- 支持多账户多权限访问管理

## 权限管理

在 `app.py` 的 `PASSWORDS` 字典中配置密码及对应权限：

```python
PASSWORDS = {
    "mima123": {
        "允许访问": True,       # 是否允许登录
        "有效期至": "20270601",  # 过期后拒绝访问
        "拒绝提示": "系统维护中",
        "到期提示": "链接已失效",
        "资产查看": False,       # 是否可查看总资产、可用资金
        "持仓查看": True,       # 是否可查看持仓明细
    },
}
```

| 权限项   | 为 True 时                           | 为 False 时              |
| -------- | ------------------------------------ | ------------------------ |
| 资产查看 | 显示总资产、可用资金、持仓市值、盈亏 | 仅显示持仓市值和今日盈亏 |
| 持仓查看 | 显示持仓明细列表                     | 隐藏持仓面板和 Tab       |

## 安全说明

- API 接口采用每日动态密钥，密钥规则为 `s_t_o_c_k_` + 当日日期（YYYYMMDD）的 MD5 值
- 展示端与交易端物理隔离，不直接接触交易软件
- 支持按密码粒度控制资产和持仓数据的可见性
- 建议部署时修改默认密钥规则和密码
